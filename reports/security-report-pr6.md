\# Отчёт по практической работе №6



\## Цель работы



Провести код-ревью проекта с целью выявления уязвимостей из OWASP Top 10, обновить список реальных и ложных срабатываний, предложить и реализовать меры устранения найденных уязвимостей.



\## Реальные уязвимости



| № | Уязвимость | Источник обнаружения | Риск | Исправление |

|---|---|---|---|---|

| 1 | Хранение паролей в открытом виде | Код-ревью, SAST | Высокий | Добавлено хэширование паролей с использованием BCrypt |

| 2 | Логирование пароля пользователя | Код-ревью | Высокий | Пароль удалён из логов |

| 3 | Возможность указать роль при регистрации | Код-ревью | Средний | При регистрации роль принудительно устанавливается как STUDENT |

| 4 | Открытый CORS | Код-ревью, DAST | Средний | CORS ограничен допустимыми origin, method и header |

| 5 | SSRF через /api/proxy | DAST, код-ревью | Высокий | Добавлена проверка URL, запрет private/local адресов и allowlist хостов |

| 6 | SSRF через /api/courses/import | DAST, код-ревью | Высокий | Добавлена проверка URL через общий валидатор |

| 7 | XXE при разборе XML | Код-ревью | Высокий | Отключены DTD и внешние сущности |

| 8 | Отсутствие защитных HTTP-заголовков | DAST | Средний | Добавлен фильтр SecurityHeadersFilter |

| 9 | Открытая H2 Console, actuator и stacktrace | Код-ревью | Средний | H2 Console отключена, actuator ограничен, stacktrace скрыт |

| 10 | Устаревшая зависимость dom4j 1.6.1 | SCA | Высокий | Зависимость обновлена до org.dom4j:dom4j:2.1.4 |



\## Ложные или условные срабатывания



| № | Срабатывание | Обоснование |

|---|---|---|

| 1 | Buffer Overflow | Приложение написано на Java, прямой переполнения буфера не подтверждено; срабатывание вызвано ответами 500 на некорректные payload |

| 2 | SQL Injection на /api/courses/search | По результатам код-ревью прямой конкатенации SQL не обнаружено; ошибка 500 вызвана некорректным входным значением |

| 3 | Unexpected Content-Type | Не является самостоятельной критичной уязвимостью, но учтено при настройке заголовков |

| 4 | Server Error response code | Само по себе является симптомом некорректной обработки ошибок, но не отдельной эксплуатационной уязвимостью |



\## Итог



В рамках работы были проанализированы результаты SAST, SCA и DAST, проведён ручной код-ревью по OWASP Top 10, выявлены реальные уязвимости и внесены исправления в код проекта.
